As you might remember, the mixing of validated and unavlidated data never sat well with me. However I am more and more convinced that we should embrace it. The client's explicit wish to do so (and the fact that no other client saw this differently), is a sign that I might have I overestimated the importance of seperating the two data types. Combining both data types in one report has the following advantages:

  • Navigation is easier to understand:
    No need to think about validated / unvalidated while choosing a date
  • The date picker is easier to use:
    More dates are available because more dates have data (either validated or unvalidated).
  • Information is easier to find:
    Because a report contains both validated and unvalidate data there are fewer empty states
  • Comparison between validated and unvalidated is easier:
    Switching between the two is instant, requiring as little from the short term memory as possible.

But what if I only want to see validated data?

I tried to keep this example as simple as possible. Assuming that all data will be be validated once, viewing a report with only validated data requires the user to pick a date far enough back in time (2016 in this example). But offering only validated data (i.e. hiding unvalidated data), would be easy. A simple checkbox 'only show validated data' would suffice. Providing both options would make it a little more complex, but still very doable.

This report contains both validated and unvalidated data. Calculations might seems incorrect. To only view unvalidated data, please check 'only show unvalidated data'.
